## 주요 Findings (심각도 순)

### P0 (즉시 수정 권장)

1) `eventFilter` 위임 helper에서 `super()` 호출 방식 오류 가능
- 근거:
  - `smartclipboard_app/ui/mainwindow_parts/ui_ops.py:280`
  - `smartclipboard_app/ui/mainwindow_parts/ui_ops.py:283`
  - `smartclipboard_app/ui/mainwindow_parts/ui_ops.py:301`
  - `smartclipboard_app/legacy_main_src.py:893`
- 상세:
  - `event_filter_impl()`은 클래스 메서드가 아닌 모듈 함수인데, 내부에서 `super().eventFilter(...)`를 호출하고 있음.
  - 이 패턴은 실행 시 `super(): no arguments` 예외를 유발할 수 있음.
- 영향:
  - 드래그 외 일반 이벤트 경로에서 `eventFilter`가 실패할 수 있어 DnD/테이블 이벤트 안정성 저하.
- 권장 조치:
  - `super().eventFilter(...)`를 `QMainWindow.eventFilter(self, source, event)` 또는 명시적 베이스 호출로 교체.
  - 해당 경로를 직접 타는 단위/통합 테스트 1개 이상 추가.

---

### P1 (높음)

2) 시그널 스냅샷 회귀 테스트가 분할된 helper 모듈을 추적하지 못함
- 근거:
  - `tests/test_signal_snapshot.py:11`
  - `scripts/refactor_signal_snapshot.py:9`
  - `scripts/refactor_signal_snapshot.py:15`
- 상세:
  - 스냅샷 생성이 `legacy_main_src.py` 내 `class MainWindow` 블록의 `.connect(` 라인만 수집함.
  - 현재는 `init_ui`/`init_menu` 연결 코드가 `smartclipboard_app/ui/mainwindow_parts/*.py`로 이동되어 커버리지가 급감.
- 영향:
  - 실제 시그널 연결 회귀가 발생해도 스냅샷 테스트가 탐지하지 못할 가능성 큼.
- 권장 조치:
  - 스냅샷 대상을 `legacy_main_src.py + mainwindow_parts/*.py`로 확장.
  - 또는 AST 기반으로 `MainWindow` 위임 메서드가 참조하는 helper 함수까지 추적.

3) `preflight_local.py`의 `py_compile` 대상이 신규 helper 모듈을 직접 포함하지 않음
- 근거:
  - `scripts/preflight_local.py:57`
  - `scripts/preflight_local.py` 내 `py_compile` 파일 목록(legacy/core 위주)
- 상세:
  - 기본 preflight는 payload 빌드 단계로 간접 검증되지만, `--skip-payload-build` 사용 시 helper 모듈 문법/임포트 오류를 놓칠 수 있음.
- 영향:
  - 로컬 빠른 검증에서 구조 분할 파일의 오류 누락 가능.
- 권장 조치:
  - `py_compile` 목록에 `smartclipboard_app/ui/mainwindow_parts/*.py` 추가.
  - 또는 `python -m compileall smartclipboard_app` 형태로 전환.

### P2 (중간)

4) `mainwindow_parts`의 `*` import 사용으로 정적 안정성/가독성 저하
- 근거:
  - `smartclipboard_app/ui/mainwindow_parts/theme_ops.py:8`
  - `smartclipboard_app/ui/mainwindow_parts/ui_ops.py:8`
  - `smartclipboard_app/ui/mainwindow_parts/menu_ops.py:8`
  - `smartclipboard_app/ui/mainwindow_parts/table_ops.py:8`
- 상세:
  - `from PyQt6.QtCore import *` 등 와일드카드 import가 광범위하게 사용됨.
- 영향:
  - 네임 충돌/사용 심볼 추적 어려움, 리팩토링 시 런타임 오류 탐지 지연.
- 권장 조치:
  - 명시 import로 점진 전환(최소한 각 모듈 상단의 핵심 타입만 명시).

5) 개발 가이드 문서가 최신 분할 구조를 충분히 안내하지 않음
- 근거:
  - `claude.md:14` (core/ui 우선 반영 원칙)
  - `claude.md:32` (preflight 권장)
  - `README.md:311` (legacy 로더/소스 설명 중심)
- 상세:
  - 현재 `MainWindow` 핵심 로직 일부가 `ui/mainwindow_parts/`로 이동했지만, 해당 경로/규칙이 문서에 명시되어 있지 않음.
- 영향:
  - 후속 작업자가 `legacy_main_src.py`만 수정하는 실수 또는 helper-legacy 불일치 유발 가능.
- 권장 조치:
  - `claude.md`에 `MainWindow 분할 영역 수정 규칙` 섹션 추가.
  - `README.md` 아키텍처 섹션에 `ui/mainwindow_parts/` 반영.
